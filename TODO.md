### Режимы работы (гибрид сценариев 1 и 3)

Управляется галочкой **«Авто-рулетка»** в админке (`auto_enabled` в SQLite).

| `auto_enabled` | Кто открывает стол | Кто крутит |
|----------------|-------------------|------------|
| **Вкл** | Первая ставка `!рулетка` в чате | Таймер сбора (60 сек) |
| **Выкл** | Кнопка «Открыть ставки» в админке | Кнопка «Крутить» в админке |

**Машина состояний** (общая для обоих режимов):

```
IDLE → OPEN → SPIN → COOLDOWN → IDLE
```

| Состояние | Поведение |
|-----------|-----------|
| **IDLE** | Ставки не принимаются (ручной режим) или ждут первую ставку (авто) |
| **OPEN** | Принимаются `!рулетка ...` (не более **одной ставки на игрока**), тикает таймер (в авто-режиме) |
| **SPIN** | Ставки закрыты, один спин, расчёт и выплата победителям |
| **COOLDOWN** | Глобальный откат 3–5 мин, новые ставки отклоняются |

**Авто-режим (`auto_enabled = true`):**
1. Стол пуст, cooldown истёк.
2. Игрок пишет `!рулетка 100 на 5` → ставка принята, раунд → `OPEN`.
3. Бот: «Игрок запустил рулетку! У остальных есть 60 секунд, чтобы сделать свои ставки!»
4. Через 60 сек: «Ставки больше не принимаются!» → `SPIN` → выплаты → `COOLDOWN`.

**Ручной режим (`auto_enabled = false`):**
1. `!рулетка` в `IDLE` → «Стол закрыт. Ждите запуска от стримера».
2. Админ жмёт «Открыть ставки» в админке → `OPEN`.
3. Чат ставит в течение окна (таймер настраиваемый, по умолчанию дольше — до 5 мин).
4. Админ жмёт «Крутить» (или таймер истекает) → `SPIN` → `COOLDOWN`.

**Сообщения в чат:** не отвечать на каждую успешную ставку — только открытие стола, закрытие ставок, результат спина. **Исключение:** отклонённая повторная ставка (игрок должен сразу понять, что второе сообщение не принято).

**Приём ставки в `OPEN` (порядок проверок):**
1. Игрок уже есть в `roulette_bets` текущего раунда? → отказ, баллы не списывать.
2. Баланс ≥ ставки, валидный синтаксис, лимиты.
3. Списать с игрока → добавить в `roulette_bank` → записать ставку в `roulette_bets`.

### Банк рулетки (`roulette_bank`)

Отдельная казна модуля. В UI и чате — **«Банк рулетки»** / **«Казна»** (слово «казино» не использовать).

**Поток баллов:**

```
[Баланс игрока] ──(ставка)──► [roulette_bank] ──(выигрыш)──► [Баланс игрока]
```

| Переменная | Значение | Где хранится |
|------------|----------|--------------|
| `roulette_bank` | Текущий баланс казны | SQLite `roulette_meta` |
| `MIN_BANK_TO_START` | `5000` | `settings.py` |
| `BANK_RESET_AMOUNT` | `50000` | `settings.py` (для `!рулетка_сброс`) |

**Правила:**

1. **Приём ставки:** сумма мгновенно списывается с баланса игрока и прибавляется к `roulette_bank`.
2. **При проигрыше:** ставка **остаётся в банке** — банк растёт.
3. **Валидация старта раунда:** если `roulette_bank < MIN_BANK_TO_START` → игра заблокирована:
   `«Рулетка отключена: в казне менее 5000 баллов»`.
4. **Выплаты:** только из `roulette_bank`, не «из воздуха».

**Алгоритм выплат в конце раунда (`SPIN`):**

1. Сгенерировать число 0–36, определить победителей.
2. Для каждого победителя вычислить **идеальный выигрыш** по формуле типа ставки.
3. `Общая_Сумма_Выплат` = сумма всех идеальных выигрышей.

**Если `Общая_Сумма_Выплат ≤ roulette_bank`:**
- каждый победитель получает полную выплату;
- `roulette_bank -= Общая_Сумма_Выплат`.

**Если `Общая_Сумма_Выплат > roulette_bank` (банкротство банка):**
- `Коэффициент = roulette_bank / Общая_Сумма_Выплат`
- `Реальная_Выплата = ⌊Идеальный_Выигрыш × Коэффициент⌋` (округление вниз)
- каждый победитель получает урезанную выплату;
- `roulette_bank = 0`;
- игра блокируется до пополнения;
- в чат: «Выплаты урезаны: в казне не хватило средств».

### Колесо

- Европейская рулетка: числа **0–36** (37 секторов).
- **0** — зелёное; не входит ни в красное/чёрное, ни в чётное/нечётное, ни в малые/большие.
- Красные: `1, 3, 5, 7, 9, 12, 14, 16, 18, 19, 21, 23, 25, 27, 30, 32, 34, 36`
- Чёрные: `2, 4, 6, 8, 10, 11, 13, 15, 17, 20, 22, 24, 26, 28, 29, 31, 33, 35`

### Типы ставок

| Тип | Синтаксис | Условие выигрыша | Идеальный выигрыш |
|-----|-----------|------------------|-------------------|
| **Прямая** | `!рулетка <ставка> на <число>` | Выпало это число | `ставка × (36 / N)` |
| **Множество** | `!рулетка <ставка> на <n1,n2,...>` | Выпало любое из чисел | `ставка × (36 / N)` |
| **Цвет** | `!рулетка <ставка> красное` / `черное` | Выпало число нужного цвета | `ставка × 1` |
| **Чётность** | `!рулетка <ставка> четное` / `нечетное` | Чётное / нечётное (не 0) | `ставка × 1` |
| **Половина** | `!рулетка <ставка> малые` / `большие` | 1–18 / 19–36 | `ставка × 1` |

- **N** — количество чисел через запятую.
- Вся сумма ставки участвует в формуле целиком (не дробится на числа).
- **Лимит:** не более **18** чисел → `«Нельзя ставить более чем на 18 чисел одновременно!»`
- **Одна ставка на игрока за раунд:** в состоянии `OPEN` у каждого `user_id` не больше одной принятой ставки. Повторное сообщение (`!рулетка 100 малые`, затем `!рулетка 100 большие`) **отклоняется** без списания баллов.

> **Множитель 35 vs 36:** в раннем примере было `300 × 35 = 10 500`, в формуле — `× 36 / N`. При реализации зафиксировать один вариант (по формуле ТЗ: `× 36 / N`).

### Примеры команд

```
!рулетка 100 на 5
!рулетка 500 на 1,13,27
!рулетка 300 красное
!рулетка 300 черное
!рулетка 200 четное

Повторная ставка тем же игроком в том же раунде (даже другой тип) — **не принимается** (см. валидацию).

### Справка для игроков (`!рулетка правила`)

Команда без ставки — показывает краткую инструкцию. Работает в любом состоянии раунда, баллы не списывает.

**Триггер:** точное совпадение `!рулетка правила` (без лишних аргументов).

**Ответ бота (одной строкой):**
```
!рулетка <сумма> [<число> | красное/черное | четное/нечетное | малые/большие | 0] — один раунд на чат, одна ставка на игрока, ~60 сек на приём.
```

### Примеры ответов бота

**Открытие раунда (авто):**
```
Игрок запустил рулетку! У остальных есть 60 секунд, чтобы сделать свои ставки!
```

**Результат — выигрыш на цвет:**
```
Выпало 7 красное! Игрок, вы выиграли 100 баллов.
```

**Результат — выигрыш на число:**
```
Выпало 1! Игрок, вы сорвали куш и выиграли 10 800 баллов!
```

**Результат — проигрыш:**
```
Выпало 0 зелёное! Никто не угадал. Ставки остались в казне.
```

**Банкротство банка:**
```

```

**Повторная ставка в том же раунде:**
```
Игрок, вы уже сделали ставку в этом раунде. Дождитесь следующего спина.
```

### Валидация

| Проверка | Сообщение |
|----------|-----------|
| Стол закрыт (ручной режим) | `Стол закрыт. Ждите запуска от стримера.` |
| Cooldown | `Рулетка отдыхает. Подождите N мин.` |
| Банк < MIN_BANK_TO_START | `Рулетка отключена: в казне менее 5000 баллов.` |
| Баланс игрока < ставки | `Недостаточно баллов. Нужно N, у тебя M.` |
| > 18 чисел | `Нельзя ставить более чем на 18 чисел одновременно!` |
| Число вне 0–36 | `Число должно быть от 0 до 36.` |
| Дубликаты в списке | `Числа в ставке не должны повторяться.` |
| Уже ставил в этом раунде (`OPEN`) | `Вы уже сделали ставку в этом раунде. Дождитесь следующего спина.` |

Проверка дубликата: перед списанием баллов искать `user_id` в `roulette_bets` текущего `round_id`. В SQLite — уникальный индекс `(round_id, user_id)`.

### Админка (`obs/admin.html` — вкладка «Рулетка»)

**Настройки:**
- ☑ Авто-рулетка (`auto_enabled`)
- Таймер сбора ставок (сек)
- Cooldown после спина (сек)

**Статус:**
- Состояние: IDLE / OPEN / SPIN / COOLDOWN + таймер
- Банк рулетки: текущий `roulette_bank`
- Список ставок текущего раунда (ник, тип, сумма)
- Итог последнего спина

**Кнопки (ручной режим, `auto_enabled = false`):**
- «Открыть ставки»
- «Крутить»
- «Пополнить банк» (опционально, дубль `!рулетка_пополнить`)
- «Отменить раунд» с возвратом ставок (опционально)

### API

```
GET  /api/roulette          — auto_enabled, state, bank, timer, bets[], last_result
PUT  /api/roulette          — { auto_enabled, ... }

### SQLite

Таблица `roulette_meta` (одна строка `id = 1`):
- `bank INTEGER` — `roulette_bank`
- `auto_enabled INTEGER`
- `state TEXT` — IDLE / OPEN / SPIN / COOLDOWN
- `round_opened_at REAL`, `cooldown_until REAL`, …

Таблица `roulette_bets` (ставки текущего/последнего раунда):
- `round_id`, `user_id`, `user_name`, `amount`, `bet_type`, `bet_payload` (JSON)
- **UNIQUE (`round_id`, `user_id`)** — одна ставка на игрока за раунд

Миграция: `bot/db/migrations/m00N_roulette.py`.
